# Manexa

SaaS-платформа для управления строительными проектами: проекты и объекты, задачи, документооборот с генерацией документов, финансы, сметы, склад материалов, график работ, согласования и чат — плюс отдельная админ-панель платформы с биллингом по подпискам.

## Архитектура

Мультитенантный монолит на Next.js 14 (App Router). Каждая компания-клиент изолирована по `companyId` на уровне всех запросов; поверх компаний работает панель платформы (`/platform`) для владельца SaaS.

```
Next.js 14 (App Router, API Routes) ── Prisma 5 ── PostgreSQL (через PgBouncer)
        │                                │
        ├─ NextAuth (JWT, 2FA TOTP)      ├─ Redis (BullMQ: очередь экспорта документов)
        ├─ Socket.IO (чат, realtime)     ├─ MinIO (S3: файлы, чеки, вложения, фото)
        └─ PM2: app + export-worker      └─ Gotenberg (PDF-рендеринг)
```

## Технологический стек

| Слой | Технологии |
|---|---|
| Frontend | Next.js 14, React 18, TypeScript, TailwindCSS, Radix UI (shadcn-паттерн), lucide-react |
| Backend | Next.js API Routes, Prisma 5.22, PostgreSQL 15, PgBouncer |
| Аутентификация | NextAuth (credentials, JWT), TOTP 2FA (otplib), bcrypt |
| Файлы | MinIO (S3-совместимое хранилище), стриминг через приложение |
| Очереди | Redis + BullMQ (асинхронный экспорт документов Excel/PDF) |
| Документы | docx/xlsx генерация по шаблонам, Gotenberg (PDF), брендирование (печать/подпись) |
| Realtime | Socket.IO (чат, индикатор печати, упоминания) |
| Процессы | PM2 (app + document-export-worker), Docker Compose (инфраструктура) |

## Модули продукта

### Рабочие
- **Дашборд** — приветствие и сводка дня, финансовые KPI (получено за месяц, дебиторка, кредиторка), мои задачи, требующие внимания проекты, ожидающие согласования, лента активности.
- **Проекты** — таблица с финансовой сводкой (доходы/расходы/прибыль), приоритетами и сроками; карточка проекта с вкладками: Обзор (KPI + структура расходов), Реквизиты клиента (автозаполнение по ИНН), Команда, Чат.
- **Задачи** — статусы, приоритеты, дедлайны, подзадачи, комментарии с упоминаниями, multi-select исполнителей.
- **График работ** — диаграмма Ганта по этапам, чек-листы, фото-отчёты, экспорт в Excel.
- **Согласования** — маршруты утверждения документов, комментарии, вложения, история.
- **Чат** — общий и по проектам, вложения файлов, эмодзи, упоминания @сотрудник и #проект, WebSocket.
- **Глобальный поиск** — ⌘K: проекты, задачи, документы.

### Финансы и документы
- **Финансы проекта** — реестр доходов (счета из генератора, отметка оплаты) и расходов (флаг «по смете», чеки-фото, кто купил); карточки: Договор / Выставлено / Получено / Не оплачено / Потрачено / К оплате.
- **Сметы** — многотабличные сметы с позициями, итогами и экспортом.
- **Материалы (склад)** — номенклатура, движения (приход/расход/корректировка), остатки, минимальные остатки, стоимость склада.
- **Документы** — генератор: договоры, КП, счета, УПД, КС-2/КС-3, акты; DOCX-шаблоны с тегами автозаполнения (реквизиты компании и клиента), сквозная нумерация, печать и подпись на экспорт, асинхронный экспорт в Excel/PDF через очередь, ЭДО-выгрузка (XML УПД).
- **Отчёты** — финансовые, по проектам, пользователям и документам; экспорт в Excel.

### Администрирование компании
- **Пользователи** — роли OWNER / ADMIN / MANAGER / USER, RBAC на уровне API и навигации.
- **Настройки** — профиль, реквизиты и брендирование компании (печать/подпись — только OWNER/ADMIN), уведомления о сроках, 2FA (TOTP с QR), смена пароля.
- **Уведомления** — in-app центр уведомлений + баннеры-анонсы от платформы.

### Панель платформы (`/platform`, для владельца SaaS)
- **Дашборд** — MRR/ARR, ARPU, платящие компании, конверсия в платные, отток, помесячная динамика; списки «истекают за 7 дней» и «просрочены/заблокированы».
- **Компании** — карточка: реквизиты (редактирование), подписка, использование (хранилище с лимитом тарифа, последний вход), пользователи (сброс пароля, деактивация, вход от имени пользователя с аудитом), CRM (ответственный менеджер, теги, заметки).
- **Биллинг** — тарифы (CRUD, лимиты по пользователям/проектам/хранилищу), подписки (триал/активна/просрочена/заблокирована), ручная фиксация оплат с продлением периода.
- **Платежи** — реестр с фильтрами и итогами, CSV-выгрузка для бухгалтерии, возвраты (только админ).
- **Анонсы** — рассылка баннеров всем компаниям (уровни инфо/предупреждение/важно, период показа, скрытие пользователем).
- **Аудит** — журнал всех действий платформенных пользователей (кто, что, когда, IP).
- **Доступ** — роли PLATFORM_ADMIN / PLATFORM_MANAGER, обязательная 2FA, сессия 8 часов с принудительным перелогином.

## Безопасность

- Изоляция данных по `companyId` во всех API-маршрутах.
- RBAC: проверка прав (`checkPermission`) на каждом эндпоинте; платформенные маршруты — отдельный контур (`checkPlatformPermission`).
- 2FA (TOTP) для любых аккаунтов; для платформенных ролей — укороченная сессия.
- Автовыход по бездействию (1 час) с редиректом на вход.
- Rate-limit на вход и экспорт документов; блокировка компаний при просрочке подписки (страница `/suspended`).
- Файлы отдаются стримингом через приложение с проверкой доступа (MinIO наружу не торчит).
- Полный аудит действий платформы (`PlatformAuditLog`).

## Установка

### Требования
- Node.js 20+
- Docker + Docker Compose

### Быстрый старт

```bash
git clone https://github.com/SultanSulimanSerj/Manexa-Claude.git
cd Manexa-Claude
npm install

# Инфраструктура: PostgreSQL, PgBouncer, Redis, MinIO, Gotenberg
cp .env.example .env        # заполнить секреты
docker compose up -d

# База данных
npx prisma generate
npx prisma db push
npm run platform:seed-plans           # тарифы
npm run platform:create-admin -- admin@example.com "Пароль123" "Имя"  # админ платформы

# Запуск
npm run dev                 # http://localhost:3000
```

### Продакшен

```bash
npm run build
npm run start:pm2           # PM2: manexa (app) + manexa-export-worker (очередь экспорта)
```

Деплой на сервер одной командой (rsync → build → `prisma db push` → PM2 restart → healthcheck):

```bash
./scripts/deploy.sh
```

## Переменные окружения

Основные (полный список — в `.env.example`):

| Переменная | Назначение |
|---|---|
| `DATABASE_URL` | PostgreSQL (рекомендуется через PgBouncer) |
| `NEXTAUTH_URL`, `NEXTAUTH_SECRET` | NextAuth |
| `REDIS_URL` | Очередь экспорта документов (BullMQ) |
| `MINIO_*` | S3-хранилище файлов |
| `GOTENBERG_URL` | PDF-рендеринг |
| `ALLOW_PUBLIC_REGISTRATION` | Публичная регистрация компаний (вкл/выкл) |

## Полезные команды

```bash
npm run db:push              # синхронизация схемы Prisma (без миграций)
npm run db:studio            # Prisma Studio
npm run worker:export        # воркер экспорта документов (вне PM2)
npm run test                 # smoke-тесты
npm run test:upd             # golden-тест генерации УПД
npm run platform:seed-plans  # тарифы платформы
```

